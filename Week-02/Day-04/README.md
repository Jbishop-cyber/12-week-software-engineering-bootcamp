# 📌 Overview

Today focused on learning **Program Design**, an important step before writing code. Instead of immediately programming a solution, I learned how to analyze a problem, break it into smaller tasks, choose the appropriate data structures, and design the overall flow of a program using algorithms and pseudocode.

The day's activities emphasized planning first and coding second, helping me develop a more structured approach to software development.

---

## 📖 Topics Covered

* What is Program Design?
* Why planning before coding is important
* Algorithms
* Pseudocode
* Flowcharts
* Input → Process → Output (IPO)
* Top-Down Design
* Modular Programming
* Choosing appropriate data structures
* Validation Rules
* Business Logic

---

## 💡 Key Takeaways

* Good software starts with good planning.
* Every program can be broken into smaller, reusable functions.
* Algorithms describe *what* a program should do before deciding *how* to code it.
* Pseudocode provides a language-independent way to plan solutions.
* Business rules define how a system behaves, while validation ensures user input is correct.
* Choosing the right data structure makes programs easier to build, maintain, and extend.
