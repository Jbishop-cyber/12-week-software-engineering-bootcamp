# Week 1 Project Summary

## Student Grade Report System

This project serves as the capstone project for Week 1 and combines all the Python concepts learned throughout the week into one complete application.

## What I Learned

### Breaking Programs into Functions

Instead of writing one long program, I divided it into smaller functions, each responsible for a single task.

Examples include:

* Getting the number of students
* Calculating grades
* Calculating remarks
* Calculating the class average
* Displaying student reports
* Displaying the class summary

This made the program easier to read, debug, and maintain.

---

### Input Validation

I learned how to repeatedly ask the user for input until a valid value is entered.

Examples:

* Number of students must be greater than zero.
* Student scores must be between 0 and 100.

This was achieved using `while` loops.

---

### Conditional Statements

I used `if`, `elif`, and `else` statements to determine:

* Student grades
* Performance remarks

This helped me understand how programs make decisions based on different conditions.

---

### Loops

I used:

* `for` loops to process multiple students.
* `while` loops to validate user input.

This showed me how loops reduce repetitive code.

---

### Functions

Functions allowed me to organize my code into reusable blocks.

I learned:

* How to define functions.
* How to pass arguments.
* How to return values.
* How different functions work together to complete a larger program.

---

### Basic List Usage

Although lists will be covered in detail later, I used a list to store student scores so I could calculate:

* Highest score
* Lowest score
* Average score

using Python's built-in functions.

---

### Built-in Functions

This project introduced me to several useful built-in functions:

* `sum()`
* `max()`
* `min()`
* `len()`

These simplified many calculations.

---

## Challenges I Faced

During development I encountered several challenges, including:

* Indentation errors
* Returning values from functions
* Passing arguments correctly
* Understanding variable scope
* Distinguishing between local and global variables
* Organizing the overall flow of the program

Working through these challenges helped me better understand how Python programs are structured.

---

## Key Takeaways

* Functions make programs easier to organize.
* Input should always be validated.
* Large problems should be divided into smaller tasks.
* Planning program flow before coding reduces errors.
* Clean, readable code is easier to maintain and debug.

This project marks the successful completion of my first week of learning Python and provides a strong foundation for more advanced topics in the coming weeks.
